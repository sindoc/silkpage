Chapter 1. Dataset: U.S. Treasury Holdings by China, 1990–2220

Table of Contents
  • 1. Schema
  • 2. Key Milestones
  • 3. Sample Records (first 5 historical)

The dataset comprises 113 observations stored as Apache Parquet (us_debt_held_by_china.parquet). 87 records are historical (sourced from U.S. Treasury TIC data); 26 are base-case projections with explicit assumption notes.

1. Schema

FieldTypeDescription
datedatetime64[us]Observation or projection date (ISO 8601)
amount_held_by_chinaint64USD billions, rounded to nearest billion
event_descriptionstrEvent context, estimation note, or projection assumption
projection_flagbooltrue = projected, false = historical

2. Key Milestones

  • 1994-01-01 — $35B: China pegs RMB to USD at 8.28; structural accumulation begins.

  • 2001-12-11 — $82B: China accedes to WTO; export boom drives rapid Treasury accumulation.

  • 2008-09-15 — $585B: Lehman collapse; flight-to-quality surge.

  • 2013-11-01 — $1,317B: All-time peak. China holds 7.9% of total US public debt.

  • 2022-02-24 — $1,055B: Russia-Ukraine invasion; Western sanctions on Russian central bank reserves change China's strategic calculus.

  • 2022-12-31 — $867B: Record annual decline of $201B.

  • 2024-12-31 — $759B: Near 14-year low; 42% below peak.

Chapter 2. Hive DDL and Analytical SQL

Table of Contents
  • 1. Hive External Table DDL
  • 2. SQL Query Catalogue

Twelve analytical SQL queries were generated for Hive, Spark SQL, Presto/Trino, and DuckDB. The source files are china_debt_hive.ddl and china_debt_queries.sql.

1. Hive External Table DDL

-- =============================================================================
-- Hive DDL: u.s. debt held by china
-- Generated by load_china_debt_into_singine.py
-- Parquet source: us_debt_held_by_china.parquet
-- =============================================================================

-- EXTERNAL TABLE (point at the directory containing the parquet file)
CREATE EXTERNAL TABLE IF NOT EXISTS china_us_debt (
    date                  DATE        COMMENT 'Observation or projection date',
    amount_held_by_china  BIGINT      COMMENT 'USD billions, rounded to nearest billion',
    event_description     STRING      COMMENT 'Historical event or projection assumption note',
    projection_flag       BOOLEAN     COMMENT 'true = projected, false = historical'
)
STORED AS PARQUET
LOCATION 'hdfs:///data/finance/china_us_debt/'
TBLPROPERTIES (
    'parquet.compress'='SNAPPY',
    'comment'='U.S. national debt held by China 1990-2220, historical + projected'
);


-- Verify load
SELECT COUNT(*) AS total_rows,
       SUM(CASE WHEN projection_flag = false THEN 1 ELSE 0 END) AS historical_rows,
       SUM(CASE WHEN projection_flag = true  THEN 1 ELSE 0 END) AS projected_rows
FROM china_us_debt;

2. SQL Query Catalogue

QueryPurpose
Q1Full historical time series (sorted)
Q2Peak holding (all-time maximum)

Q4Decade-level band summary (min/max/avg)

-- =============================================================================
-- Analytical SQL: U.S. Debt Held by China
-- Compatible with: Hive, Spark SQL, Presto/Trino, DuckDB
-- =============================================================================


-- ── Q1: Full historical time series (sorted) ──────────────────────────────────
SELECT date,
       amount_held_by_china,
       event_description
FROM   china_us_debt
WHERE  projection_flag = false
ORDER  BY date;


-- ── Q2: Peak holding (all-time maximum) ──────────────────────────────────────
SELECT date,
       amount_held_by_china,
       event_description
FROM   china_us_debt
WHERE  projection_flag = false
ORDER  BY amount_held_by_china DESC
LIMIT  1;


-- ── Q3: Year-over-year change (historical only) ───────────────────────────────
SELECT
    curr.date                                             AS date,
    curr.amount_held_by_china                             AS amount_usd_bn,
    prev.amount_held_by_china                             AS prev_amount_usd_bn,
    (curr.amount_held_by_china - prev.amount_held_by_china)
                                                          AS yoy_change_bn,
    ROUND(
        100.0 * (curr.amount_held_by_china - prev.amount_held_by_china)
              / prev.amount_held_by_china,
        2
    )                                                     AS yoy_pct_change
FROM   china_us_debt curr
JOIN   china_us_debt prev
    ON  YEAR(curr.date) = YEAR(prev.date) + 1
    AND MONTH(curr.date) = 1
    AND MONTH(prev.date) = 1
WHERE  curr.projection_flag = false
   AND prev.projection_flag = false
ORDER  BY curr.date;


-- ── Q4: Decade-level summary (historical) ────────────────────────────────────
SELECT
    FLOOR(YEAR(date) / 10) * 10          AS decade,
    MIN(amount_held_by_china)            AS min_bn,
    MAX(amount_held_by_china)            AS max_bn,
    ROUND(AVG(amount_held_by_china), 0)  AS avg_bn,
    COUNT(*)                             AS observations
FROM   china_us_debt
WHERE  projection_flag = false
GROUP  BY FLOOR(YEAR(date) / 10) * 10
ORDER  BY decade;


-- ── Q5: Key crisis / event observations ─────────────────────────────────────
SELECT date,
       amount_held_by_china,
       event_description
FROM   china_us_debt
WHERE  projection_flag = false
  AND  event_description IS NOT NULL
  AND  (
         date IN (
             DATE '2001-12-11',  -- WTO accession
             DATE '2008-09-15',  -- Lehman collapse
             DATE '2010-10-01',  -- Peak as largest holder
             DATE '2013-11-01',  -- All-time peak
             DATE '2015-08-11',  -- RMB devaluation
             DATE '2018-07-06',  -- Trade war begins
             DATE '2022-02-24',  -- Russia invasion / sanctions shock
             DATE '2022-12-31'   -- Largest annual decline on record
         )
       )
ORDER  BY date;


-- ── Q6: Inflection points — largest single-period drops ──────────────────────
SELECT
    curr.date,
    curr.amount_held_by_china,
    (curr.amount_held_by_china - prev.amount_held_by_china) AS change_bn,
    curr.event_description
FROM   china_us_debt curr
JOIN   china_us_debt prev

Chapter 4. Interactive HTML Visualization

Table of Contents
  • 1. Features
  • 2. Events Context Parquet

The file china_debt_timeline.html is a fully self-contained interactive visualization (no server required) generated by generate_timeline_viz.py.

1. Features

  • Dark-theme Chart.js 4.4 line chart with gradient fill under the historical series (solid blue) and a dashed amber projection line.

  • Color-coded scatter event markers (six categories) overlaid on the chart — fully clickable.

  • Three view range controls: 1990–2030 / 1990–2060 / Full Timeline (to 2220).

  • Category filter buttons: Crisis · Policy · Trade · Geopolitical · Records · All.

  • Horizontal scrollable event timeline strip with significance-height bars.

  • Slide-out detail panel with full narrative, economic indicators grid, significance dots, source note, and prev/next navigation.

  • Six KPI cards: all-time peak ($1.317T), current ($759B), decline (−42%), record drop (−$201B), % of total debt (2.1%), trend (REDUCING).

2. Events Context Parquet

22 key events are stored in china_debt_events_context.parquet (and china_debt_events_context.json) with the following enriched fields:

  • event_id, category, headline

  • description — 200–400 word narrative per event

  • significance — 1–5 rating

  • impact_direction — INCREASE / DECREASE / STABLE / PEAK

  • related_indicators_json — era-specific economic indicators

  • source_note, projection_flag
