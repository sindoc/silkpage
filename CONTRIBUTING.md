# Contributing

SilkPage accepts contributions, but non-trivial code contributions should be
covered by contributor terms that allow the project to continue using the code
under both GPL and commercial license arrangements.

## Inbound Licensing Policy

For contributions that affect the dual-licensed software scope:

- Small fixes may be accepted under the project license terms alone.
- Non-trivial patches should be accompanied by a signed contributor agreement.
- Contributors must only submit work they have the right to license.
- Contributors must identify any third-party code copied or adapted into a
  patch.

## Required Documents

- Individual contributors: `legal/CLA-Individual.md`
- Company-sponsored contributions: `legal/CLA-Corporate.md`

## Scope Split

Different repository areas may be governed differently:

- Core software and build logic may fall into the GPL plus commercial scope.
- Documentation may remain under a documentation license.
- Website copy, templates, and visual assets may remain under content licenses
  unless explicitly relicensed.

See `LICENSES/README.md` for the current intended partition.

## Submission Workflow

1. Open an issue or pull request describing the change.
2. Confirm whether the change is in the dual-licensed software scope.
3. If required, sign the appropriate contributor agreement.
4. Preserve existing copyright and attribution notices.
5. Add or update tests and documentation when behavior changes.

This file is project process guidance and not legal advice.
